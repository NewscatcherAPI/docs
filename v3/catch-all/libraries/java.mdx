can use the REST API directly with your preferred HTTP client.

### Using the REST API

Here's a basic example using Java's built-in HTTP client:

```java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class CatchAllExample {
    public static void main(String[] args) throws Exception {
        HttpClient client = HttpClient.newHttpClient();

        // Create a job
        String requestBody = """
            {
                "query": "Tech company earnings this quarter",
                "context": "Focus on revenue and profit margins"
            }
            """;

        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://catchall.newscatcherapi.com/catchAll/submit"))
            .header("x-api-key", "YOUR_API_KEY")
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(requestBody))
            .build();

        HttpResponse<String> response = client.send(
            request,
            HttpResponse.BodyHandlers.ofString()
        );

        System.out.println(response.body());
    }
}
```

## Alternative approaches

Until the Java SDK is available, consider:

- Using the [Python SDK](/v3/catch-all/libraries/python) or
  [TypeScript SDK](/v3/catch-all/libraries/typescript) if your architecture
  supports it.
- Making direct HTTP requests to the
  [REST API endpoints](/v3/catch-all/endpoints/create-job).
- Using a Java HTTP client library like OkHttp or Apache HttpClient.
