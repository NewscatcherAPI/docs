Use the CatchAll app to integrate structured web research directly into your
Make.com automation scenarios.

## Before you start

Before you begin, make sure you have:

- Make.com account
- CatchAll API key (obtain from
  [platform.newscatcherapi.com](https://platform.newscatcherapi.com))

## Install the app

The CatchAll app is available by invitation. Use the link below to install it in
your Make.com organization.

<Steps>
  <Step title="Access installation page">
    Open the [CatchAll app invitation link](https://eu1.make.com/app/invite/39a1ccee5332419f4260db8eddb4739b) in your browser.
  </Step>

<Step title="Install app">
  On the installation page, click **Install** to add CatchAll to your Make.com
  organization.
</Step>

  <Step title="Verify installation">
    The app is now available in your scenario editor. When you search for modules, CatchAll appears with an "Invited" badge.
  </Step>
</Steps>

## Configure connection

Set up your API key to authenticate CatchAll modules.

<Steps>
  <Step title="Add CatchAll module">
    Click **+** in any scenario, search NewsCatcher CatchAll, and select a module.
  </Step>

<Step title="Create connection (first time only)">
  When you add your first CatchAll module, a connection dialog appears
  automatically: 
  
  1. Click **Create a connection**.
  2. Enter a connection name.
  3. Paste your CatchAll API key into the **API Key** field.
  4. Click **Save**.
  
</Step>

  <Step title="Reuse connection">
    The connection is now available for all CatchAll modules in your scenarios. Select it from the **Connection** dropdown in each module.
  </Step>
</Steps>

## Available modules

CatchAll modules are organized into Actions and Searches.

### Actions

| Module            | Description               | Endpoint                             |
| ----------------- | ------------------------- | ------------------------------------ |
| `submit-query`    | Create a new CatchAll job | `/catchAll/submit`                |
| `create-monitor`  | Schedule recurring jobs   | `/catchAll/monitors/create  `     |
| `enable-monitor`  | Resume a paused monitor   | `/catchAll/monitors/{id}/enable`  |
| `disable-monitor` | Pause a running monitor   | `/catchAll/monitors/{id}/disable` |

### Searches

| Module                 | Description                    | Endpoint                         |
| ---------------------- | ------------------------------ | -------------------------------- |
| `search-job-id`        | Check job processing status    | `/catchAll/status/{job_id}`    |
| `pull-data`            | Retrieve completed job results | `/catchAll/pull/{job_id}`      |
| `get-monitor-schedule` | List monitor execution history | `/catchAll/monitors/{id}/jobs` |
| `pull-monitor-results` | Get aggregated monitor records | `/catchAll/monitors/pull/{id}` |

<Note>
  Actions perform operations, while Searches retrieve data. Choose the
  appropriate category when adding modules to your scenario.
</Note>

## Create your first scenario

Build a scenario that submits a CatchAll job, waits for completion, and sends
results via email.

![Complete Make.com workflow from trigger to email notification](/images/integrations/make-job-workflow-complete.png)

<Note>
  Want to skip manual setup? Import the [public scenario template](#) 
  directly into your Make.com account.
</Note>

### Add modules

<Steps>
  <Step title="Add trigger">
    Add **Tools > Basic trigger** to start the scenario manually.
  </Step>

<Step title="Submit job">
  Add **NewsCatcher CatchAll > submit-query**: 
  
  - **Connection**: Select your CatchAll connection. 
  - **Query**: Enter a natural language question. 
  - **Context** (optional): Add focus instructions.
  - **Schema** (optional): Specify output format template.
</Step>

<Step title="Initial wait">
  Add **Tools > Sleep**: 
  
  - **Delay**: `60` Wait 60 seconds before checking status for the first time.
</Step>

<Step title="Add polling loop">
  Add **Flow Control > Repeater**: 
  
  - **Initial value**: `1` 
  - **Repeats**: `20`
</Step>

<Step title="Check status">
  Add **NewsCatcher CatchAll > search-job-id**:
  
  - **Connection**: Select your CatchAll connection 
  - **Job ID**: Map `job_id` from the **submit-query** module.
</Step>

  <Step title="Add Router">
    Add **Flow Control > Router** to handle different job states.
  </Step>
</Steps>

### Configure routes

The Router evaluates job status and directs the flow accordingly.

<Steps>
  <Step title="Route 1: Job complete">
    Configure the first route:
    
    1. Click the route path to open filter settings.
    2. **Label**: `Complete`
    3. **Set as fallback**: No
    4. **Condition**:
       - Click **Set up a filter**.
       - From the mapping panel, expand **search-job-id** module output.
       - Navigate into `steps` array > index `[5]` > drag `completed` field to condition
       - **Operator**: Select **Boolean operators: Equal to**
       - **Value**: `true`
    5. Click **Save**
  </Step>

<Step title="Route 2: Job failed">
  Add a second route: 
  
  1. Click **+** on the Router to add another route. 
  2. **Label**: `Failed` 
  3. **Set as fallback**: No 
  4. **Condition**: - From `steps` array > index `[6]` > drag `completed` field 
     - **Operator**: **Boolean operators: Equal to** 
     - **Value**: `true` 
  5. Click **Save**
</Step>

  <Step title="Route 3: Still processing">
    Add a third route:
    
    1. Click **+** on the Router to add another route.
    2. **Label**: `Processing`
    3. **Set as fallback**: Yes
    4. No condition needed - this catches all other cases.
  </Step>
</Steps>

### Add route actions

<Steps>
  <Step title="Complete route actions">
    On the **Complete** route path:
    
    1. Add **Flow Control > Break** (stops the Repeater).
    2. Add **NewsCatcher CatchAll > pull-data**:
       - **Connection**: Select your connection.
       - **Job ID**: Drag `job_id` from **submit-query** module.
    3. Add **Email > Send an Email** and configure it to send results to your email address.
  </Step>

<Step title="Failed route actions">
  On the **Failed** route path: 
  
  1. Add **Flow Control > Break**.
  2. Add **Email > Send an Email** to notify yourself of the failure.
</Step>

  <Step title="Processing route actions">
    On the **Processing** route path:
    
    Add **Tools > Sleep**:
    - **Delay**: `60`
    
    The Repeater automatically starts the next iteration after this sleep.
  </Step>
</Steps>

## Test the scenario

<Steps>
  <Step title="Run scenario">
    Click **Run once** at the bottom of the editor.
  </Step>

<Step title="Monitor execution">
  Watch the scenario execute in real-time. The **Repeater** will poll job status
  every 60 seconds until the job completes or fails.
</Step>

  <Step title="Check email">
    When complete, you'll receive an email with the job results.
  </Step>
</Steps>
